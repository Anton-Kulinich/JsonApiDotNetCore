LoadCurrentRelationships(TResource, RelationshipAttribute)

Before assigning new relationship values (UpdateAsync), we need to attach the current database values of the relationship to the dbContext, else it will not perform a complete-replace which is required for one-to-many and many-to-many.

For example: a person p1 has 2 todo-items: t1 and t2. If we want to update this todo-item set to t3 and t4, simply assigning p1.todoItems = [t3, t4] will result in EF Core adding them to the set, resulting in [t1 ... t4]. Instead, we should first include [t1, t2], after which the reassignment p1.todoItems = [t3, t4] will actually make EF Core perform a complete replace. This method does the loading of [t1, t2].
